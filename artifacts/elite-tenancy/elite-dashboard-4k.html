✅ Stripe Webhook Raw Body
Webhook mounted BEFORE express.json() with express.raw() — HMAC-SHA256 signature verified correctly. This is the #1 most common Stripe integration mistake. Solved perfectly.
✅ WebSocket Token Auth
Short-lived 30s single-use token pattern solves WS auth without cookie forwarding dependency. Crypto-random 32 bytes. Consumed on first use. Pruned every 60s.
✅ Conversation Participant Guard
ws/server.ts line 94 checks conv.landlordId === userId || conv.tenantId === userId before persisting message. Prevents users messaging conversations they don't belong to.
✅ Message Content Validation
WS server validates content: not empty, not over 4000 chars. Trim applied before persist. Prevents blank messages and storage abuse.
✅ Idempotent Migrations
All 3 migrations use IF NOT EXISTS + DO $$ BEGIN...EXCEPTION WHEN duplicate_object. Safe to run multiple times on server restart.
✅ Replaced Connection on Reconnect
clients.set(userId, ws) replaces existing WS connection and closes the old one. Prevents zombie connections accumulating in memory.
✅ Stripe Returns 200 on Handler Error
Webhook handler catches errors but still returns 200 to Stripe. Prevents unnecessary retries flooding the endpoint when a handler bug occurs.
✅ pool.connect().finally().release()
getOrCreateStripeCustomer uses try/finally to release the pool connection. No connection leaks even on error path.
✅ Tier Validation
payments.ts line 51: explicit allowlist check ["standard","professional","premium"].includes(tier) before touching Stripe. Prevents SSRF via invalid price IDs.
✅ Upsert for Conversations
onConflictDoUpdate on (landlord_id, tenant_id, listing_id) UNIQUE constraint. startConversation() is fully idempotent — no duplicate conversations created.
✅ TypeScript Strict Throughout
Route handlers typed as Promise<void>. Return types explicit. No implicit any. Drizzle inferSelect used for DB types. Full type safety chain from DB → API → React.
✅ CORS Explicit Allowlist
No origin reflection. ALLOWED_ORIGINS env var is a comma-separated explicit list. Unknown origins rejected with 403. Standard OWASP recommendation.
✅ pnpm Workspace Monorepo
Shared @workspace/db and @workspace/api-client-react packages. Single source of truth for DB schema and API types. Zero drift between frontend and backend contracts.
✅ REST Fallback for Messages
ChatContext.tsx falls back to REST POST /messages/.../send if WS is not OPEN. No message loss when connection drops. Graceful degradation implemented correctly.
🛡️
XSS Prevention in Aria Chat
HTML is escaped before markdown render in AriaChat.tsx. Server-side session history prevents prompt injection from user-controlled input. Gemini responses treated as untrusted until sanitised.
🛡️
No SQL Injection Surface (Drizzle)
All database queries use Drizzle ORM's parameterised query builder. The only raw SQL is in migrations (no user input). Zero injection surface in ORM layer.
🛡️
Rate Limiting on AI Endpoint
Aria chat is rate-limited to 20 requests/minute. Prevents abuse of Gemini API quota and denial-of-service via LLM cost amplification.
⚠️
WS Rate Limiting Missing
No per-connection message rate limit on WebSocket. Authenticated users can send unlimited messages per second. Add a token bucket (60 msg/min) per userId in wsTokenStore or a separate Map.
⚠️
No Stripe Idempotency Key
Repeated POST /payments/landlord/checkout calls create new Checkout Sessions each time. A network retry or double-click causes duplicate charges. Add idempotencyKey option to stripe.checkout.sessions.create.
